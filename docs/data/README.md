# Data Model

All data lives in BigQuery project `prisme-wamba-2026`, dataset `prisme_dataset`.

---

## Tables

### products_selected

Raw Open Food Facts data, filtered and deduplicated. Source of truth for the pipeline.

| Column | Type | Description |
|--------|------|-------------|
| `ean` | STRING | EAN barcode (primary key) |
| `product_name` | STRING | Product display name |
| `brands` | STRING | Brand(s), comma-separated |
| `categories` | STRING | Category hierarchy, comma-separated |
| `ingredients_text` | STRING | Raw ingredients list |
| `nutriscore_grade` | STRING | a / b / c / d / e |
| `quantity` | STRING | Net weight or volume |
| `packaging` | STRING | Packaging type |
| `image_url` | STRING | Source image URL (Open Food Facts CDN) |

### clip_embeddings

CLIP vector embeddings per product, used for visual similarity search.

| Column | Type | Description |
|--------|------|-------------|
| `ean` | STRING | |
| `run_id` | STRING | |
| `run_date` | DATE | |
| `embedding` | ARRAY<FLOAT64> | 512-dimensional L2-normalized CLIP vector |

Query pattern:
```sql
SELECT base.ean, distance
FROM VECTOR_SEARCH(
  TABLE `prisme_dataset.clip_embeddings`,
  'embedding',
  (SELECT [0.12, -0.34, ...] AS embedding),
  top_k => 10,
  distance_type => 'COSINE'
)
```

## Cloud Storage

Bucket: `gs://prisme-assets` (public read, `allUsers objectViewer`)

```
prisme-assets/
  originals/          # full-resolution product images
    {ean}.jpg
  thumbnails/
    128/{ean}.jpg     # 128x128 px
    256/{ean}.jpg     # 256x256 px
    512/{ean}.jpg     # 512x512 px
```

Public CDN base: `https://storage.googleapis.com/prisme-assets/`

## Scoring formulas

```
text_score     = weighted sum of field completeness (max 100)
visual_score   = resolution*0.4 + sharpness*0.4 + centration*0.2
catalog_score  = text_score*0.6 + visual_score*0.4
```

All scores are integers in `[0, 100]`. Missing images yield `visual_score = 0`.
